# Questions About Using GitHub in the Classroom

## Computer Science Classroom with Firia Labs' CodeX

### Getting Started & Setup
1. How do you create a GitHub account and set up a repository for storing CodeX projects?
2. What is the difference between a public and private repository, and which is more appropriate for student coding projects?
3. How can students connect their CodeX projects to GitHub for version control?
4. What is the recommended workflow for committing CodeX MicroPython code to a repository?

### Version Control & Workflow
5. Why is version control important when writing programs for the CodeX, and what problems does it solve?
6. How do commits help you track the history of a project as your program evolves?
7. What is a branch, and how could branching help students experiment with different versions of a CodeX program without breaking working code?
8. How would you use a pull request in a student coding project, and what does the review process look like?
9. When might you want to "roll back" to a previous commit, and how would you do that?

### Collaboration
10. How can two students collaborate on the same CodeX project using GitHub without overwriting each other's work?
11. What are merge conflicts, and how do you resolve them when two students edit the same file?
12. How can a teacher use GitHub to review and give feedback on student code submissions?
13. What is a GitHub Organization, and how could a teacher use one to manage all student repositories for a class?
14. How does forking a repository differ from cloning it, and when would each be appropriate for a student?

### Real-World Connections
15. How do professional software developers use GitHub in their daily work, and how does learning it in CS class prepare students for careers in tech?
16. How could open-source projects on GitHub serve as inspiration or learning resources for CodeX programming?
17. What is a README file, and why is writing a good README an important professional skill?

## College Composition Concurrent Enrollment Class

### Introduction to GitHub for Writers
1. What is GitHub, and how is it similar to tools you may already use for document collaboration (like Google Docs)?
2. How can version control benefit the writing process, similar to saving drafts of an essay?
3. What does it mean to "commit" a change to a document, and how is that different from just saving a file?
4. How could using GitHub make the peer review and revision process more transparent and organized?

### Writing Process & Revision
5. How can commit history serve as a record of your writing process and revision decisions throughout a project?
6. How might you use branches to draft different versions of an introduction or argument before choosing the best one?
7. How does GitHub encourage more intentional, incremental revisions compared to editing a single document?
8. What does a meaningful commit message look like for a writing project, and why does it matter?

### Collaboration & Peer Review
9. How could pull requests be used as a structured peer review tool, where classmates suggest edits and leave comments?
10. What are the benefits and challenges of collaborating on a shared document using GitHub compared to a shared Google Doc?
11. How could a class collaboratively build and contribute to a shared document, such as a course anthology or writing guide, using GitHub?
12. How do you leave a constructive, specific comment on someone's writing within a GitHub pull request?

### Academic Integrity & Attribution
13. How does GitHub's commit history support academic integrity by documenting who wrote what and when?
14. How could instructors use GitHub to verify that a submitted piece of writing reflects a student's own revision process?
15. What ethical considerations exist around using GitHub for academic work, such as keeping drafts private versus public?

### Transferable Skills
16. How might learning GitHub in a writing class prepare students for collaborative writing or documentation work in professional settings?
17. In what ways is writing documentation (like a README) for a project similar to writing a technical report or professional memo?
18. How can the skills of clear, concise commit messaging translate to professional communication skills more broadly?
